# 要件定義書: ユーザー認証機能

## はじめに

本機能は、Gleanアプリケーションにおけるユーザー認証機能を提供する。Firebase Authenticationを利用し、Google認証およびメールアドレス/パスワード認証の2つの認証方式をサポートする。認証済みユーザーのみが記事の登録・管理機能にアクセスできるようにし、パーソナライズされた記事コレクションを安全に管理できるようにする。

## プロダクトビジョンとの整合性

CLAUDE.mdに記載されている設計原則「認証やデータの登録はfirebaseを使用する」「Google認証とメールアドレスの認証ができる」に直接対応する基盤機能である。

## 要件

### 要件1: Google認証によるログイン

**ユーザーストーリー:** ユーザーとして、Googleアカウントでログインしたい。手軽にアプリを使い始められるようにするためである。

#### 受け入れ基準

1. WHEN ユーザーが「Googleでログイン」ボタンをクリック THEN システムはGoogleのOAuth認証画面にリダイレクトする
2. WHEN ユーザーがGoogleアカウントを選択し認証を完了 THEN システムはFirebase Authenticationを通じてユーザーセッションを作成し、アプリのメイン画面に遷移する
3. IF ユーザーが初めてGoogleログインする場合 THEN システムはFirestoreにユーザープロファイルを自動作成する
4. WHEN Google認証が失敗またはキャンセルされた場合 THEN システムはログイン画面に戻り、適切なエラーメッセージを表示する

### 要件2: メールアドレス/パスワードによるサインアップ

**ユーザーストーリー:** ユーザーとして、メールアドレスとパスワードで新規アカウントを作成したい。Googleアカウントを使わずにアプリを利用できるようにするためである。

#### 受け入れ基準

1. WHEN ユーザーがメールアドレスとパスワードを入力し「サインアップ」ボタンをクリック THEN システムはFirebase Authenticationでアカウントを作成する
2. IF パスワードが8文字未満の場合 THEN システムは「パスワードは8文字以上で入力してください」とエラーを表示する
3. IF メールアドレスが既に登録済みの場合 THEN システムは「このメールアドレスは既に登録されています」とエラーを表示する
4. WHEN アカウント作成が成功 THEN システムはメール確認リンクを送信し、確認待ち画面を表示する
5. WHEN ユーザーがメール確認リンクをクリック THEN システムはメール確認済みステータスに更新し、ログイン可能状態にする

### 要件3: メールアドレス/パスワードによるログイン

**ユーザーストーリー:** ユーザーとして、登録済みのメールアドレスとパスワードでログインしたい。自分の記事コレクションにアクセスするためである。

#### 受け入れ基準

1. WHEN ユーザーがメールアドレスとパスワードを入力し「ログイン」ボタンをクリック THEN システムはFirebase Authenticationで認証を行い、成功時にメイン画面へ遷移する
2. IF メールアドレスまたはパスワードが誤っている場合 THEN システムは「メールアドレスまたはパスワードが正しくありません」とエラーを表示する
3. IF メール確認が未完了の場合 THEN システムは「メールアドレスの確認が完了していません。確認メールを再送しますか？」と表示する

### 要件4: ログアウト

**ユーザーストーリー:** ユーザーとして、アプリからログアウトしたい。セキュリティを確保するためである。

#### 受け入れ基準

1. WHEN ユーザーがログアウトボタンをクリック THEN システムはFirebaseセッションを破棄し、ログイン画面に遷移する
2. WHEN ログアウト完了後 THEN システムはクライアント側の認証状態をクリアする

### 要件5: 認証状態の管理

**ユーザーストーリー:** ユーザーとして、ログイン状態が維持されてほしい。毎回ログインし直す手間を省くためである。

#### 受け入れ基準

1. WHEN ユーザーがアプリを再度開いた場合 THEN システムはFirebaseの認証状態永続化により自動的にログイン状態を復元する
2. WHEN 認証トークンが期限切れの場合 THEN システムはFirebaseが自動的にトークンをリフレッシュする
3. IF ユーザーが未認証の状態で保護されたページにアクセスした場合 THEN システムはログインページにリダイレクトする

### 要件6: パスワードリセット

**ユーザーストーリー:** ユーザーとして、パスワードを忘れた場合にリセットしたい。アカウントへのアクセスを回復するためである。

#### 受け入れ基準

1. WHEN ユーザーが「パスワードを忘れた方」リンクをクリック THEN システムはパスワードリセット画面を表示する
2. WHEN ユーザーが登録済みメールアドレスを入力し送信 THEN システムはFirebaseを通じてパスワードリセットメールを送信する
3. IF 未登録のメールアドレスが入力された場合 THEN システムはセキュリティのため「メールアドレスが登録されている場合、リセットメールを送信しました」と表示する（存在有無を漏らさない）

## 非機能要件

### コードアーキテクチャとモジュール性
- **単一責任の原則**: 認証ロジックは`src/composables/useAuth.ts`に集約する
- **モジュラー設計**: Firebase初期化、認証状態管理、UIコンポーネントを分離する
- **明確なインターフェース**: 認証コンポーザブルは明確なAPIを公開する（`login`, `logout`, `signup`, `resetPassword`, `currentUser`, `isAuthenticated`）

### パフォーマンス
- ログイン・サインアップ処理は3秒以内に完了すること
- 認証状態の復元はアプリ起動時に1秒以内に完了すること
- Firebase SDKはツリーシェイキングを活用し、必要なモジュールのみをバンドルすること

### セキュリティ
- パスワードは最低8文字とする
- Firebase Authenticationの組み込みセキュリティ機能を活用する
- メール確認を必須とする（メール/パスワード認証の場合）
- パスワードリセット時にメールアドレスの存在有無を漏らさない
- Firestoreセキュリティルールで認証済みユーザーのみデータアクセスを許可する

### 信頼性
- Firebase Authenticationのサービス可用性に依存する
- オフライン時は適切なエラーメッセージを表示する
- 認証エラー時は具体的なエラーメッセージでユーザーをガイドする

### ユーザビリティ
- ログイン/サインアップフォームはバリデーションエラーをリアルタイムで表示する
- Google認証は1クリックでログイン可能とする
- モバイル対応のレスポンシブデザインとする
- ローディング状態を適切に表示する
