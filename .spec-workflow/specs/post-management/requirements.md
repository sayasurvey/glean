# 要件定義書 - 投稿一覧・登録機能

## はじめに

Gleanの中核機能である「投稿一覧・登録機能」を実装します。認証済みユーザーが技術ブログやXの投稿などのURLを登録し、OGP画像と概要をタイル形式で一覧表示する機能です。URLからOGP情報を自動取得し、Gemini APIで記事概要を自動生成します。タグ付けとキーワード検索によって、記事を効率的に整理・検索できます。

## プロダクトビジョンとの整合性

このissue（#5）はGleanの主要機能を実現します：
- 技術情報のキュレーション（URL登録・OGP表示）
- Gemini APIによるAI概要生成
- タグ・キーワードによる検索・フィルタリング

## 要件

### 要件1: 記事・投稿の登録

**ユーザーストーリー:** 認証済みユーザーとして、技術ブログやXの投稿のURLを登録したい。後で簡単に見返せるようにするため。

#### 受け入れ基準

1. WHEN ユーザーが有効なURLを入力して登録ボタンを押す THEN システムはURLからOGP情報（タイトル、OGP画像URL、説明）をFirebase Cloud Functionsまたはサーバーサイドプロキシ経由で取得する
2. WHEN OGP情報の取得が成功 THEN システムはタイトル・URL・OGP画像URL・説明・タグ・ユーザーIDをFirestoreの`posts`コレクションに保存する
3. WHEN OGP情報の取得が失敗 THEN システムはURLとユーザーが入力したタイトルのみで登録し、失敗をユーザーに通知する
4. WHEN 記事が登録される THEN システムはGemini APIを使用して記事の概要テキストを非同期で生成し、Firestoreに保存する
5. WHEN ユーザーがタグを入力 THEN システムは入力補完で既存タグを候補表示し、選択または新規作成したタグを記事に紐づけて保存する
6. WHEN 同じURLが既に登録済み THEN システムは重複登録を警告する
7. WHEN URLの形式が無効 THEN システムはバリデーションエラーを表示する

### 要件2: 記事一覧のタイル表示

**ユーザーストーリー:** 認証済みユーザーとして、登録した記事をOGP画像付きのタイル形式で一覧表示したい。視覚的に記事を把握できるようにするため。

#### 受け入れ基準

1. WHEN ユーザーがメインページにアクセス THEN システムは自分が登録した記事をOGP画像・タイトル・概要・タグ付きのタイル形式で表示する
2. WHEN タイルをクリック THEN システムは元の記事/投稿を新しいタブで開く
3. WHEN 記事が20件を超える THEN システムは無限スクロールまたはページネーションで追加読み込みする
4. IF OGP画像が存在しない THEN システムはデフォルトのプレースホルダー画像を表示する
5. WHEN 記事一覧が読み込み中 THEN システムはスケルトンローダーを表示する
6. WHEN 登録済み記事が0件 THEN システムは「まだ記事が登録されていません」という空状態メッセージを表示する

### 要件3: 検索・フィルタリング機能

**ユーザーストーリー:** 認証済みユーザーとして、タイトル・タグ・概要からキーワード検索したい。目的の記事を素早く見つけるため。

#### 受け入れ基準

1. WHEN ユーザーが検索バーにキーワードを入力 THEN システムはタイトルと概要から部分一致する記事をリアルタイムにフィルタリングして表示する
2. WHEN ユーザーがタグチップをクリック THEN システムはそのタグが付いた記事のみを表示する
3. WHEN 検索結果が0件 THEN システムは「該当する記事がありません」と表示する
4. WHEN 検索バーをクリア THEN システムは全記事の一覧表示に戻る
5. WHEN 複数のタグを選択 THEN システムはすべてのタグを含む記事のみを表示する（AND検索）

### 要件4: 記事の削除

**ユーザーストーリー:** 認証済みユーザーとして、不要になった記事を削除したい。一覧を整理するため。

#### 受け入れ基準

1. WHEN ユーザーが記事の削除ボタンをクリック THEN システムは確認ダイアログを表示する
2. WHEN ユーザーが削除を確認 THEN システムはFirestoreから記事を削除し、一覧を更新する
3. WHEN 削除が成功 THEN システムは「記事を削除しました」と通知する
4. IF 削除しようとする記事が自分のものでない THEN システムは削除を拒否する（Firestoreセキュリティルール）

### 要件5: タグ管理

**ユーザーストーリー:** 認証済みユーザーとして、記事に関連する技術タグを付けて整理したい。カテゴリ別に記事を管理できるようにするため。

#### 受け入れ基準

1. WHEN ユーザーが記事登録時にタグを入力 THEN システムは既存タグの入力補完候補を表示する
2. WHEN ユーザーが新しいタグ文字列を確定 THEN システムは新規タグを作成して記事に紐づける
3. WHEN ユーザーが既存タグを選択 THEN システムはそのタグを記事に追加する
4. WHEN タグをクリック THEN システムはそのタグでフィルタリングする（要件3-2と連携）

### 要件6: Gemini APIによる概要自動生成

**ユーザーストーリー:** 認証済みユーザーとして、登録した記事の内容をAIで自動要約してほしい。記事の内容を素早く把握できるようにするため。

#### 受け入れ基準

1. WHEN 記事が登録される THEN システムはバックグラウンドでGemini APIにOGPの説明または記事URLを送信し概要を生成する
2. WHEN 概要生成が成功 THEN システムは生成された概要テキストをFirestoreの記事ドキュメントに保存する
3. WHEN 概要生成が失敗 THEN システムはエラーをログに記録し、概要フィールドを空のまま保存する（記事登録はブロックしない）
4. WHEN 概要生成中 THEN システムはタイルに「概要生成中...」というプレースホルダーを表示する

## 非機能要件

### コードアーキテクチャとモジュール性
- **単一責任の原則**: 各ファイルは単一の明確な目的を持つ
- **モジュラー設計**: コンポーネント、コンポーザブル、サービスは分離・再利用可能にする
- **依存関係管理**: モジュール間の相互依存を最小限にする
- **型安全性**: TypeScript strictモードに準拠し、すべての型を明示的に定義する

### パフォーマンス
- 初回記事一覧表示は2秒以内
- タイル一覧のスクロールは60fps維持
- OGP情報の取得は非同期で行いUIをブロックしない
- Firestoreクエリは適切なインデックスを使用する

### セキュリティ
- Firestoreセキュリティルールで自分のデータのみ読み書き可能にする
- Gemini APIキーはサーバーサイド（Nuxt server routes）で管理し、クライアントに露出しない
- OGPフェッチはサーバーサイドプロキシ経由でCORSを回避する
- ユーザー入力（タグ、タイトル）のサニタイズ

### 信頼性
- Firestore操作失敗時のエラーハンドリングとユーザーへのフィードバック
- Gemini API失敗時は概要なしで記事登録を継続する
- ネットワークエラー時の適切なエラーメッセージ表示

### ユーザビリティ
- レスポンシブデザイン（モバイル・タブレット・デスクトップ対応）
- ローディング状態のスケルトンUI表示
- 操作成功・失敗時のトースト通知
- 空状態の適切なメッセージとCTA（Call to Action）
